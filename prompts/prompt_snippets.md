# 提示词（可直接复用）

把提示词当成你和 AI 的协议。短句适合临时输入，专业版适合复制粘贴。

## 通用占位符模板（建议先填这个）

```text
【目标/功能】
【用户/场景】
【验收标准】
【限制/边界】
【影响范围】
【相关文件/模块（若已知）】
```

## 新需求（短句）

```text
先找入口和相似实现，给文件路径。
```

```text
给 MVP（最小可用版本）方案，别改代码。
```

```text
按小步实现，一次只改一个点。
```

```text
给最小验证步骤。
```

```text
列出本次改动的回归点（可能影响的功能）。
```

```text
先给最小计划，再实现第一步。
```

## Bug 修复（短句）

```text
列出可能原因和文件路径。
```

```text
给最小修复方案，避免大改。
```

```text
给回归点（可能影响的功能）和验证步骤。
```

```text
失败就回到干净版本再试。
```

```text
只改最小关键路径。
```

## 问题处理（短句）

```text
请只给一个可执行的小步骤，并说明验证方式。
```

```text
请缩小范围，只分析最相关的 2-3 个文件。
```

```text
请给最小方案与风险点。
```

## 能力提升（短句）

```text
请像代码审查一样指出问题，并给改进建议。
```

```text
请给出同功能的两种实现，并对比取舍。
```

```text
请给最小测试用例并解释断言。
```

## 新需求（专业版，可复制）

```text
你是我的架构工程师。请先扫描仓库，找出与【功能/模块】相关的入口、相似实现、关键数据流，输出文件路径与理由。
然后给出最小实现方案（MVP，最小可用版本）与可选增强，并列出验收标准与最小验证步骤。不要直接改代码。
```

```text
请把需求转成可执行清单，包含：关键用户流程、必须实现 vs 可延后、涉及模块/API、待确认问题（尽量少）。
```

```text
请按小步实现，每次只改一个点，完成后给出变更说明与验证步骤。
```

## Bug 修复（专业版，可复制）

```text
你是我的架构工程师。已知复现步骤与日志如下：
【复现步骤】
【当前表现】
【预期表现】
【错误日志/异常栈】
请列出 2-5 个可能原因，并指向相关文件路径。给出最小修复方案与验证步骤。
```

```text
请先分析原因再动手，避免扩散式改动。只修关键路径，补最小回归测试。
```

## 问题处理（专业版，可复制）

```text
当前阻碍是【问题描述】。请把问题收敛成一个可执行的小步骤，并说明验证方式。
```

```text
请只分析最相关的 2-3 个文件，给出最小方案与风险点。
```

## 能力提升（专业版，可复制）

```text
请对这段代码做审查，指出问题与改进建议，并解释取舍。
```

```text
请给出同功能的两种实现，并对比优缺点与适用场景。
```

```text
请生成最小测试用例，说明断言目的与覆盖边界。
```

## 启动与部署（专业版，可复制）

```text
请为本项目生成一键启动脚本，包含依赖安装、环境变量检查、服务启动与最小健康检查。
```

```text
请给出本项目的最小部署流程与回滚策略（出问题时如何撤回），输出为分步清单。
```
