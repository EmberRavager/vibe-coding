# Vibe Coding 通用方法

这是一张总地图。你不需要一次记住所有路，只需要知道：先对齐目标，再小步推进，最后验证交付。

## 你在做什么

你在用 AI 开发项目，但你不是旁观者。你负责方向与验收，AI 负责执行与补全。

## 最短路线

目标对齐 -> 找入口 -> MVP（最小可用版本） -> 小步改动 -> 验证

## 核心原则

1. 先对齐目标，再让 AI 动手
2. 先看现状，再做改动
3. 小步推进，控制风险
4. 先验证关键路径，再考虑优化
5. 变更要可追踪、有理由

## 风险分级

- 低风险：原型、探索性功能、个人项目，可更快节奏
- 高风险：生产、支付、隐私、核心链路，必须严格验证

## 标准工作流（8 步）

1. 目标对齐：一句话说明目标与验收
2. 现状扫描：入口、相似实现、数据流
3. 最小方案：先 MVP（最小可用版本）再增强
4. 小步改动：一次只改一块
5. 关键验证：最小测试或手动步骤
6. 变更说明：做了什么、为什么、如何验证
7. 回归检查：列出风险与边界
8. 再扩展：稳定后再加功能

## 常见问题

**Q：AI 写得快但不稳，怎么办？**
用小步 + 验证 + 回滚（退回到上一步）。不要让一次改动覆盖太多。

**Q：我不知道从哪里开始？**
先找入口与相似实现，入口就是起点。
