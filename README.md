# Vibe Coding 教程（从 0 到 1）

vibe coding 大体上就是敲几个字就可以让AI就可以帮你写代码，很容易上瘾，享受创造的过程。
这份教程主要是帮助大家怎么使用ai编程工具，怎么去跟ai编程软件去聊天得到自己想要的。

## 你会得到什么

- 一条从想法到代码的清晰路径
- 新需求与 Bug 修复的标准流程
- 可直接复制的提示词（短句版 + 专业版）
- 卡住时的一些解决方法

## 1 分钟理解 Vibe Coding

一句话：用 AI 写代码，但方向由你掌控，但是是方向就会偏，这就需要我们学习如何去减少偏差得到我们想要的代码。

做法很简单：先说清楚要做什么，再让 AI 走一小步，然后立刻检查有没有走对。
走错了就回到上一步，再试一次，所以现在很多程序员大部分都是ai售后程序员，每天都在和ai斗智斗勇，希望ai可以一步到位，但是都事与愿违。

## 出发前的一句话提醒

AI 很快，但方向是你的。只要你能给清楚目标和验收，它就能把大部分路铺好，但是大部分时候它都没办法一步到位，需要反复的调教。

## 一次任务通常长这样

你拿到一个需求，先问入口在哪，再要一个最小方案，然后每次只做一小步分的功能，之后自己去实际验证。
如果跑偏，再回到之前的正确的状态上重来。整个过程像走钢丝，但你永远有安全绳。

## 5 分钟上手示例（完整闭环）

场景：从 0 到 1 做一个“本地待办清单（Todo）小工具”

先填通用模板（照着填，不用追求完美）：

```text
【目标/功能】
一句话说清“要做什么”。
示例：做一个本地 Todo 小工具，支持新增、查看、完成、删除。

【用户/场景】
谁在什么情况下使用。
示例：我自己在命令行快速记任务，不依赖网络。

【验收标准】
写 3-5 条“可检查”的结果，尽量可执行、可观察。
示例：
1) 我输入一条任务后，列表里能马上看到
2) 我能看到每条任务是“未完成”还是“已完成”
3) 我把某条任务标记为已完成后，状态会变化
4) 我删除某条任务后，它不会再出现

【限制/边界】
明确不做什么，避免 AI 发散。
示例：只做本地 JSON 存储；不做登录/云同步/图形界面。

【影响范围】
会改哪些层，帮助 AI 估算工作量。
示例：CLI 参数解析、任务存储模块、基础 README 使用说明。

【相关文件/模块（若已知）】
已有项目就填路径；从 0 到 1 可写“暂无”。
示例：src/cli.ts、src/store.ts、README.md（从 0 到 1 可先留空）。
```

步骤 1：先要脚手架与目录

```text
按“单文件先跑通”的原则，给出项目初始化命令、目录结构和最小依赖。
```

步骤 2：要 MVP（最小可用版本）方案

```text
先给我最小可用方案和顺序，先做“能记录并看到任务”，再做“完成和删除”，先别改代码。
```

步骤 3：小步实现

```text
按小步实现：先做“新增任务、查看任务”并保证能跑；再做“标记完成、删除任务”；每步都告诉我改了哪些文件。
```

步骤 4：最小验证

```text
给 3 条可直接复制的验证命令，并说明预期输出。
```
