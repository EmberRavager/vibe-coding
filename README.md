# Vibe Coding 教程（从 0 到 1）

把它想象成一场陪跑：你带着想做的产品，AI 帮你实现，但你始终握着方向盘。
这份教程不是堆概念，而是一条可以从需求走到上线的路。

## 你会得到什么

- 一条从想法到代码的清晰路径
- 新需求与 Bug 修复的标准流程
- 可直接复制的提示词（短句版 + 专业版）
- 卡住时的自救策略

## 1 分钟理解 Vibe Coding

一句话：用 AI 写代码，但方向由你掌控。

做法很简单：先说清楚要做什么，再让 AI 走一小步，然后立刻检查有没有走对。
走错了就回到上一步，再试一次。

低风险可以快一点，高风险一定要稳一点。

## 适合谁

如果你是以下几类人，这份教程会很顺手：

- 想用 AI 做出自己产品的 PM
- 想提高效率但不想失控的开发者
- 想快速试错并保留可维护性的独立开发者

## 出发前的一句话提醒

AI 很快，但方向是你的。只要你能给清楚目标和验收，它就能把大部分路铺好。

## 一次任务通常长这样

你拿到一个需求，先问入口在哪，再要一个最小方案，然后只做一小步，立即验证。
如果跑偏，回到干净状态重来。整个过程像走钢丝，但你永远有安全绳。

## 5 分钟上手示例（完整闭环）

场景：新增“订单导出 CSV”

先填通用模板：

```text
【目标/功能】
【用户/场景】
【验收标准】
【限制/边界】
【影响范围】
【相关文件/模块（若已知）】
```

步骤 1：先找入口

```text
先找入口和相似实现，给文件路径。
```

步骤 2：要 MVP（最小可用版本）方案

```text
给 MVP（最小可用版本）方案，别改代码。
```

步骤 3：小步实现

```text
按小步实现，一次只改一个点。
```

步骤 4：最小验证

```text
给最小验证步骤。
```
