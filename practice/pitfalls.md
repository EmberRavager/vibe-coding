# 常见误区与纠偏

你会遇到这些坑，但只要抓住节奏，就不会失控。

## 误区 1：直接让 AI 改代码

症状：改动范围失控，目标漂移。

纠偏提示词：

```text
先给最小方案和变更范围，别改代码。
```

## 误区 2：一次改太多

症状：难以定位问题，回滚成本高。

纠偏提示词：

```text
拆成小步骤，一次只改一个点。
```

## 误区 3：不验证就交付

症状：上线后回归问题频发。

纠偏提示词：

```text
给最小验证步骤和回归点（可能影响的功能）。
```

## 误区 4：没有上下文

症状：输出泛化、不贴合项目。

纠偏提示词：

```text
请先找入口与相似实现，给文件路径与理由。
```

## 误区 5：修复失败还硬改

症状：代码越来越乱。

纠偏提示词：

```text
失败就回到干净版本再试。
```
