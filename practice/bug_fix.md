# Bug 修复（详细版）

修 Bug 的关键不是快，而是稳。先复现，再定位，最后修。

## 最短流程

复现 -> 定位假设 -> 最小修复 -> 最小验证

## 1. 复现与定义

让 AI 明确“当前表现”和“预期表现”。

输入要点：

- Bug 描述：
- 复现步骤：
- 当前表现：
- 预期表现：
- 影响范围：

## 2. 定位与假设

让 AI 给出 2-5 个可能原因，并指向文件路径。

## 2.1 收集证据

- 贴完整错误日志或异常栈
- 标出最小复现路径

## 3. 最小修复

只修关键路径，避免扩散式重构。

## 4. 验证与回归

执行最小测试或手动步骤，确认修复不影响相邻逻辑。

## 常见问题

**Q：修完又坏了？**
先回到干净版本（回滚到上一次正常状态），再尝试更小的修复。

**Q：原因不清楚？**
先列可能原因与路径，再动手。

## 可用提示词

```text
基于复现步骤和日志，列出最可能原因与最小修复方案。
```

```text
请先分析原因再动手，避免扩散式改动。只修关键路径，补最小回归测试。
```
